# Workable Terminology & Word Choice Guide

Reference for consistent terminology across all Workable product surfaces.

---

## AI Product Terminology

### Canonical Terms — Always Use These

| Term | Usage |
|---|---|
| **Workable AI** | The umbrella brand for all AI capabilities in Workable |
| **Workable Agent** | The autonomous AI recruiting agent (referred to as "the agent" in short form) |
| **Ask Workable** | The conversational AI assistant embedded in the product |
| **Smart tools** | AI-powered assistants within specific features (e.g., job description generator) |
| **Candidate Evaluator** | The AI tool that assesses candidate fit |
| **Candidate Finder** | The AI tool that surfaces matching candidates |
| **job requirements** | What a role needs — use this term, not "job description requirements" or "criteria" |
| **candidate fit** | How well a candidate matches a role — use this term |

### AI Terms to Avoid

| Don't use | Use instead |
|---|---|
| generate | create, write, find, suggest (be specific about the action) |
| assistant | use the specific product name (Ask Workable, Workable Agent) |
| copilot | (avoid entirely — Microsoft trademark) |
| AI Recruiter | Workable Agent |
| bot | agent |
| automate | (avoid — implies replacing people; use "handle", "manage", "take care of") |
| smart | (avoid as a generic modifier — use the specific feature name) |
| powered by AI | (avoid in UI — use specific capability names) |
| AI-powered | (avoid as a generic label — describe what the AI does) |

### Writing Principles for AI Features

- **Name the action, not the technology** — Tell users what will happen, not that AI is involved
- **Use active, specific verbs** — "Workable finds candidates who match your criteria" not "AI-powered candidate sourcing"
- **Don't over-promise** — Avoid claiming the AI will "find the best candidate" or "guarantee results"
- **Be transparent about AI involvement** — On first use or in sensitive contexts, disclose that AI is being used
- **Surface-specific guidance:**
  - **Buttons** — Action verb + noun: "Find candidates", "Write job description"
  - **Nav labels** — Nouns only: "Candidate Evaluator", "Ask Workable"
  - **Helper text** — One sentence; describe input and what the AI produces
  - **Empty states** — Lead with value, follow with action
  - **Notifications** — State what the agent did or found; link to result
  - **Disclosure** — Plain language; avoid legal-sounding boilerplate

## Word Choice A–Z

### Choose vs. Select
- Use **"choose"** when the user is making a decision between options that are more subjective or open-ended
- Use **"select"** in two cases:
  1. When giving device-agnostic instructions (more inclusive than "click" — works for mouse, touch, keyboard)
  2. When a user must pick one option from a defined list

✅ "Select your birthday", "Select 'Add new' to add another location"  
❌ "Click 'Add new' to add another location", "Choose your birthday"

### Edit vs. Change
- Use **"edit"** when the user is modifying existing content (text, settings, records)
- Use **"change"** only in natural language contexts, not as a CTA label

✅ "Edit profile", "Edit job"  
❌ "Change profile"

### Login vs. Log in
- **"login"** — noun or adjective: "your login", "the login page"
- **"log in"** — verb (two words): "log in to your account"

✅ "Log in to Workable", "Go to the login page"  
❌ "Login to Workable"

### My vs. Your
- Always use **"your"** — never "my" — when referring to things that belong to the user
- Workable speaks to users in second person ("your profile"), not first person from the user's perspective ("my profile")

✅ "Your profile", "Your time off balances"  
❌ "My profile", "My time off"

See also: **Perspective** below.

### Profile Photo vs. Profile Picture
- Always use **"profile photo"** — not "profile picture", "avatar", or "profile image"

✅ "Update your profile photo"  
❌ "Upload a profile picture"

### Dropdown vs. Drop-down
- Always **"dropdown"** as one word (no hyphen)

### Email
- Always lowercase: **"email"** — not "e-mail" or "Email" (unless starting a sentence)

### Resume vs. CV
- Always use **"resume"** — not "CV" (even for European audiences)
- CV = curriculum vitae; used in Europe. We default to US English throughout.

✅ "Send your resume to the hiring manager"  
❌ "Please submit your CV along with a cover letter"

### Select vs. Click
- Prefer **"select"** over "click" in user instructions — it's device-agnostic and more inclusive

### Setup vs. Set up
- **"setup"** — noun or adjective: "account setup", "integration setup"
- **"set up"** — verb (two words): "set up your account", "set up an integration"

✅ "Start integration setup", "Set up your account"  
❌ "Start integration set up", "Setup your account"

### Self- prefix
- Always hyphenate: **self-review**, **self-motivated**, **self-assessment**, **self-improvement**

✅ "self-review", "self-motivated"  
❌ "selfreview", "selfmotivated"

### View vs. See
- Use **"view"** when giving instructions to look at something
- Avoid "see" in instructional copy

✅ "View your remaining time-off balance"  
❌ "See your remaining time-off balance"

### Wifi
- Always lowercase, no spaces or hyphens: **wifi**

✅ "public wifi"  
❌ "Wi-Fi", "WIFI"

### Workable
- Use **"Workable"** or **"we"** to refer to the company or product — not "our product" or "the platform"

✅ "Workable connects to 100+ integrated software providers", "We couldn't process your request"  
❌ "Our product connects to 100+ providers"

## Perspective

Always write from **Workable's point of view**, addressing the user in second person.

- Use **"your"** for things belonging to the user
- Use **"Workable"** or **"we"** for things belonging to the product or company
- Never use **"my"** in UI copy (even though the feature might feel like "mine" from the user's perspective, we speak to users as "you")

✅ "Your profile", "Your time off balance"  
❌ "My profile", "My time off"

## Capitalization Rules for Product Terms

**Free Tools for Managers** — capitalize all four words when referring to the specific Workable product tier  
**Workable Agent** — capitalize both words  
**Ask Workable** — capitalize both words  
**Candidate Evaluator** — capitalize both words  
**Candidate Finder** — capitalize both words  

General rule: Capitalize named Workable features and products. Don't capitalize generic UI terms (button, page, tab, modal, dropdown, etc.).

## Slashes

- Avoid using slashes as a substitute for "and" or "or" — write those words out instead
- If you must use a slash in description text or dropdown options, write it **without spaces**: `withdrew/declined`
- Use slashes **with spaces** only when displaying multiple job locations: `Athens, Greece / London, UK / Berlin, Germany`

✅ "Workable is an ATS and HRIS software", "Candidate withdrew/declined interview"  
❌ "Workable is an ATS/HRIS software", "Candidate withdrew / declined interview"
